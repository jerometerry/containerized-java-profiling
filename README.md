## Profiling

### Testing Locally

Run [profile.sh](observability/profile.sh) to spin up a docker container running a Spring Boot
application to generate a synthetic load, capture CPU and memory allocation flame graphs, copy them
back to the host, then terminate the container.

Here's a [CPU](observability/cpu-flame.html) flame graph generated by running [profile.sh](observability/profile.sh).

Here's a [memory allocation](observability/alloc-flame.html) flame graph generated by running
[profile.sh](observability/profile.sh).

### Running in AWS

#### Shell Access

If you want to run these commands from the AWS console, get shell access using this command

```bash
aws ecs execute-command \
  --cluster prototype-perf-cluster \
  --task <TASK_ID> \
  --container spring-app \
  --interactive \
  --command "/bin/bash"
```

#### CPU Profile

Once you have access to the container, run the async profiler to capture a CPU flame graph.

```bash
/opt/async-profiler/bin/asprof -e itimer -d 30 -f /app/profiles/cpu-flame.html 1
```

#### Memory Profile

Once you have access to the container, run the async profiler to capture memory allocation flame graphs.

```bash
/opt/async-profiler/bin/asprof -e alloc -d 30 -f /app/profiles/alloc-flame.html 1
```
